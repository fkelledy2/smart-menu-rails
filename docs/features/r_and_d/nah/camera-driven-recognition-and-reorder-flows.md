# Camera-Driven Recognition & Reorder Flows

**Status:** Research / Strategy
**Feasibility:** Currently Feasible
**Target:** 2026+

## Vision

The camera becomes an input layer for dish recognition, reorder shortcuts, and menu education.

## Included Ideas

- Camera-based dish recognition
- Photograph dish to identify it and reorder

## Feasible Now

- Fine-tuned image classification or multimodal LLM recognition
- Assisted confirmation flows
- Reorder shortcuts linked to recognized `Menuitem` records

## Constraints

- Requires high-quality training or reference data
- Ambiguity for visually similar dishes
- Confirmation remains necessary to avoid mistakes

## Suggested R&D Path

- Start with dish recognition for menu education, not direct ordering
- Add reorder shortcuts after confidence thresholds are acceptable
- Improve models using restaurant-specific reference imagery over time

## Strategic Value

- Novel but practical camera input surface
- Helps guests discover or repeat items quickly
- Builds reusable multimodal recognition capability
