# Presence, Identity & Return-Guest Warm Start

**Status:** Research / Strategy
**Feasibility:** Currently Feasible
**Target:** 2026+

## Vision

Detect likely return guests and preload context before they actively begin ordering.

## Included Ideas

- Automatic guest detection
- Menu preload on arrival

## Feasible Now

- Logged-in session recall
- Returning-device heuristics with consent
- Venue-entry hints from app or web session behavior

## Constraints

- Privacy and consent boundaries
- Must avoid feeling creepy or overly assumptive

## Suggested R&D Path

- Focus first on performance preloading, not silent identity assumptions
- Use explicit opt-in where persistent return-guest behavior is involved
- Expand personalization only when trust is high

## Strategic Value

- Reduces startup friction
- Makes repeat visits feel faster and smarter
- Supports later personalization layers
