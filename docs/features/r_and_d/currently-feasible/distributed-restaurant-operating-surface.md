# Distributed Restaurant Operating Surface

**Status:** Research / Strategy
**Feasibility:** Currently Feasible
**Target:** 2026+

## Vision

Unify guest and staff device experiences into a shared ordering and coordination substrate.

## Included Ideas

- Distributed ordering system
- Phones replace some fixed POS or table terminals
- Staff and guests use variants of the same interface

## Feasible Now

- Staff handheld UI on the existing web stack
- Shared order and event models across customer and staff contexts
- Table and kitchen coordination via realtime updates

## Strategic Value

- Strong product unification
- Reinforces the “restaurant floor OS” framing
- Reduces dependence on specialized hardware over time

## Suggested R&D Path

- Start by unifying core order state across staff and guest surfaces
- Expand staff workflows on top of the same event model
- Use this as the operating substrate for future table, kitchen, and payment intelligence
