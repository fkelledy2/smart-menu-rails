# Staff Assistance & Proximity Response

**Status:** Research / Strategy
**Feasibility:** Currently Feasible
**Target:** 2026+

## Vision

Allow guests to summon staff digitally and route requests intelligently.

## Included Ideas

- Bluetooth waiter paging
- Nearest waiter notification
- “Need assistance” requests

## Feasible Now

- Digital assistance requests from guest devices
- Staff app notifications based on station or zone assignment
- Simple proximity approximations from staff role and location context

## Constraints

- True nearest-device Bluetooth vibration is harder in browser-only flows
- Native staff apps would improve reliability

## Suggested R&D Path

- Ship zoned or station-based waiter paging first
- Add richer routing logic later
- Introduce stronger proximity signals only if hardware or native apps justify it

## Strategic Value

- Reduces guest friction in requesting help
- Creates operational visibility for service responsiveness
- Can improve service speed without adding staff overhead
