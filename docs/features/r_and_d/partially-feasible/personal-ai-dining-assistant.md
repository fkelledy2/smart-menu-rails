# Personal AI Dining Assistant

**Status:** Research / Strategy
**Feasibility:** Partially Feasible / Constrained by Current Platforms
**Target:** 2026+

## Vision

Every guest gets an increasingly personalized dining layer that understands preferences, restrictions, and pairing behavior.

## Included Ideas

- Persistent dining profiles
- Ingredient and dietary preference filtering
- AI wine or dish pairing
- Personalized menu ranking

## Feasible Now

- Stored preference profiles
- Rules + embeddings + LLM ranking over structured menu data
- Pairing suggestions based on menu attributes and historical choices
- Session-level and account-level personalization

## Constraints

- High-quality personalization requires enough repeat-user data
- Recommendations must be explainable and trustworthy
- Sensitive preference data needs careful handling

## Future Expansion

- Portable preference profiles across multiple venues
- Faster, cheaper real-time inference everywhere

## Suggested R&D Path

- Start with explicit user preferences first
- Then layer inferred preferences gradually
- Keep recommendations transparent, editable, and reversible

## Strategic Value

- Strong long-term moat
- Can improve conversion, satisfaction, and retention
- Compounds as user data and venue coverage grow
