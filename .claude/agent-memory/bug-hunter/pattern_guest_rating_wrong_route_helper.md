---
name: Guest Rating Wrong Route Helper
description: _guest_rating_form used restaurant_ordr_guest_rating_path — correct helper is guest_rating_restaurant_ordr_path (FIXED)
type: project
---

`_guest_rating_form.html.erb` used `restaurant_ordr_guest_rating_path(restaurant, order)` but the route named helper generated by Rails for a member action `guest_rating` nested under `restaurants/ordrs` is `guest_rating_restaurant_ordr_path`. Every render of the post-checkout rating form would have raised `NoMethodError`. Fixed by correcting the path helper in the view.

**Why:** Route helpers for member actions follow the pattern `{action}_{parent_resource}_{resource}_path`, not `{parent_resource}_{resource}_{action}_path`.

**How to apply:** When adding new member-route actions, always verify the generated path helper name with `rails routes | grep action_name` before using it in views.
